# Project Scope

## Project Name

Scratch Gap Maze

## Core Problem

Many beginners learn programming logic through block-based environments such as Scratch, but struggle when moving to text-based programming languages such as Python. The transition can feel sudden because learners must deal with syntax, indentation, spelling, and error messages all at once.

## Core Question

How can visual programming support be removed gradually rather than all at once?

## Version 1 Goal

Build a web-based maze programming environment where learners begin with blocks, see the equivalent Python-style code, and eventually replace selected blocks with text while keeping the program runnable.

## Version 1 Includes

- Maze-based challenges
- Sequencing
- Loops
- Conditionals
- Simple functions
- Debugging support
- Live Python-style code view
- Hybrid block/text transition system
- Beginner-friendly error explanations
- User testing with beginner programmers

## Version 1 Excludes

- Variables
- Full Python support
- User accounts
- Backend database
- AI chatbot tutor
- Multiplayer
- Mobile app
- Teacher dashboard
- Certificates
- Multiple game types

## Main Technical Challenge

The central technical challenge is building a reliable bridge between block-based programs and Python-style text. The app should use an internal program representation so that both blocks and text snippets can be converted into the same structure and executed by the maze engine.

## Success Criteria

The project will be considered successful if:

- users can solve maze challenges using blocks
- users can view accurate Python-style code generated from their blocks
- users can convert selected blocks into text snippets
- mixed block/text programs can still run correctly
- beginner-friendly errors help users understand mistakes
- user testing provides evidence about where the tool helps and where it needs improvement
