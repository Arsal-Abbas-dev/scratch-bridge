This file records the main design decisions, technical challenges, bugs, fixes, and reflections during the development of Scratch Gap Maze.

- Opened the project in VS Code.
- Replaced the default starter screen with a simple project title and description.
- Started the README file.
- Started the development log.
- Created a scope document to prevent the project from becoming too broad.

### Design decision

I decided to keep Version 1 focused on maze-based programming challenges. The project will not attempt to support all of Python. Instead, it will support a restricted Python-style subset connected to maze commands.

### Why this matters

The main aim of the project is not to build another general coding website. The main technical challenge is to create a gradual transition between block-based programming and text-based programming.
